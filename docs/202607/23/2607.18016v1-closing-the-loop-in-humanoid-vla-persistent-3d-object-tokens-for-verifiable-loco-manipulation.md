## 摘要
视觉-语言-动作策略是通用机器人控制的有前景基础，但长时域人形行走操作要求机器人将任务对象视为在运动、接触、遮挡和恢复过程中持续存在的物理实体。我们将此问题研究为对象状态发散：用于条件化全身动作的对象状态可能与用于判断动作是否达到预期物理关系的状态不同。我们提出持久对象标记化（POT），该方法从RGB-D观测中维护角色索引的3D对象记录，并将其转换为用于全身动作专家的对象标记。实例化为POT-VLA，相同的对象记录条件化动作生成并支持几何谓词检查，从而形成一个闭环执行系统，其中对象状态既可操作又可验证。在Unitree G1上，POT-VLA在八个真实世界任务族中将匹配的直接GR00T-N1.7基线从39/80成功提升至71/80成功。在外部Being-0对齐参考中，POT-VLA在对齐服务任务上实现44/50成功，而Being-0论文报告的成功率为37/50。最大的提升出现在需要维持3D关系的任务上，表明持久以对象为中心的状态是可验证人形VLA执行的有用抽象。

## Abstract
Vision-language-action policies are a promising foundation for general robot control, but long-horizon humanoid loco-manipulation requires the robot to treat task objects as persistent physical entities across movement, contact, occlusion, and recovery. We study this problem as object-state divergence: the object state used to condition a whole-body action can differ from the state used to decide whether the action achieved the intended physical relation. We propose \emph{Persistent Object Tokenization} (POT), which maintains role-indexed 3D object records from RGB-D observations and converts them into object tokens for a whole-body action expert. Instantiated as \emph{POT-VLA}, the same object records condition action generation and support geometric predicate checks, yielding a closed-loop execution system in which object state is both actionable and verifiable. On a Unitree G1, POT-VLA improves a matched direct GR00T-N1.7 baseline from 39/80 to 71/80 successes over eight real-world task families. In an external Being-0-aligned reference, POT-VLA achieves 44/50 successes on aligned service tasks, compared with the 37/50 success reported by the Being-0 paper. The largest gains occur on tasks requiring maintained 3D relations, suggesting that persistent object-centered state is a useful abstraction for verifiable humanoid VLA execution.

## 1. 核心问题与整体含义
- **研究动机**：人形机器人长时域行走操作（loco-manipulation）中，存在**对象状态发散**问题（object-state divergence），即用于条件化动作的对象状态与用于验证动作是否达成物理关系的状态不一致。这导致抓取、放置、恢复等环节的虚假成功或错误，任务可靠性下降。
- **整体含义**：提出**持久对象标记化（POT）**方法，通过维护角色索引的共享3D对象记录，同时用于动作生成和几何验证，构建一个**可验证的闭环执行系统**。核心观点是：只有将对象状态持久化、可操作、可验证，才能提升人形VLA在真实环境中的执行可靠性。

## 2. 方法论
- **核心思想**：构建**持久对象标记（Persistent 3D Object Tokens）**，将RGB-D观测中任务相关实体转化为角色索引的3D对象记录，序列化为固定槽位的特征张量。同一记录用于：
  - 条件化全身动作预测（动作专家使用）
  - 几何谓词检查（验证动作结果）
- **关键技术细节**：
  - **POT构建**：对每个活动子任务τᵢ，通过VLM规划器或任务文件获取实体集Eᵢ。利用RGB-D观测和SAM3分割，构建对象内存Mₜ^{τᵢ}，每个记录包含：角色（TARGET/DESTINATION/SUPPORT/HANDOVER_PARTNER）、短语、2D框、3D中心、范围、置信度、可见性、关系特征（如目标-容器距离、支撑高度、机械臂-对象偏移等）。
  - **标记化模式**：默认架构使用K=8个槽，每个槽F=33个特征，包括：可见性、置信度、归一化2D框、3D中心、3D范围、可选方向、关系特征。未使用槽填充并掩码。所有度量在机器人基坐标系下归一化。
  - **动作头插入**：使用GR00T-N1.7作为动作专家。POT-VLA将对象标记投影到DiT隐藏维度（LayerNorm→Linear→GELU→Dropout→Linear），插入到动作头自注意力序列中：布局为 `[状态标记, 对象标记, 动作标记]`。对象标记结合角色嵌入和上下文嵌入，填充槽掩码输出。
  - **闭环流程**：执行一个动作块后，RGB-D刷新相同对象内存，几何谓词监督器评估物理关系（包含、支撑、接近、位移等），返回 `in_progress/done/blocked/failed/uncertain` 状态，并决定继续/重试/重新观测/重新定位/重新规划。
