## 摘要
视觉-语言-动作（VLA）模型展现出令人印象深刻的泛化能力，但往往缺乏可解释性，且难以遵循编码了空间、时序和逻辑要求的精确自然语言指令。我们提出一种层次化框架，使用信号时序逻辑（STL）作为连接高层语言理解与低层机器人执行的共享表示。高层策略利用VLM将语言指令分解为高层子任务，为每个子任务生成STL规范，并选择执行每个子任务的低层策略。STL规范将语言导出的意图转化为精确约束，低层策略选择则决定这些约束是通过STL引导的模型预测控制直接强制执行，还是在执行学习策略（针对感知复杂或接触密集行为）时进行监控。通过将STL集成到计划验证、低层策略、子任务监控和重新规划中，我们的框架使得语言导出的计划能够在运行时使用通用的形式化结构进行检查、优化和修订。我们在真实世界的桌面任务域上评估了该方法，展示了形式化规范如何提高语言条件化机器人规划的精确性、可靠性和可解释性。

## Abstract
Vision-language-action (VLA) models have shown impressive generalization, but often lack interpretability and can struggle to follow precise natural language instructions that encode spatial, temporal, and logical requirements. We propose a hierarchical framework that uses Signal Temporal Logic (STL) as a shared representation connecting high-level language understanding with low-level robot execution. A high-level policy leverages a VLM to decompose language instructions into high-level subtasks, generate STL specifications for each subtask, and choose a low-level policy for executing each subtask. The STL specifications translate language-derived intent into precise constraints, and the low-level policy selection determines whether those constraints are enforced directly through STL-guided model-predictive control or monitored during execution of a learned policy for perceptually complex, or contact-rich behaviors. By integrating STL into plan validation, low-level policy, subtask monitoring, and replanning, our framework enables language-derived plans to be checked, optimized, and revised at runtime using a common formal structure. We evaluate the approach on a real-world tabletop domain, demonstrating how formal specifications can improve the precision, reliability, and interpretability of language-conditioned robot planning.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 构建层次化框架 **STeP**，以 STL 作为共享表示，连接高层语言理解和低层机器人控制。
- System 2（VLM 负责）将语言指令分解为子任务，并生成对应的 STL 规范（包含安全、时序、空间约束）。
- System 1 根据子任务特性，选择 **STL-guided MPC**（需精确约束时）或 **预训练学习策略**（如接触复杂行为）执行，同时使用 STL 鲁棒性进行监控和失败检测。

### 关键技术细节
- **STL 信号谓词与鲁棒性**：预定义谓词库（如 Safe, Grasp, PlaceOn），原子谓词为 μ(z_t) ≥ c，鲁棒性定义为 μ(z_t)-c。使用 softmax 平滑 max/min 用于梯度优化。
- **System 2：语言到 STL 的规划**：
  1. 输入语言指令、当前观测、技能库、历史日志；
  2. VLM（GPT-4o）输出结构化的子任务序列（每个子任务含技能名、参数、约束、自然语言描述）；
  3. 静态任务规范检查（JSON 格式、技能存在性、参数类型、对象存在性等）；
  4. STL 编译器将每个子任务技能模板和参数映射为 STL 公式（嵌入全局约束如安全、时间窗口）。
- **System 1：STL 引导的执行与监控**：
  - **STL-guided MPC**：优化动作序列最小化成本函数：`sum(c(s_k,a_k)) - λ * ρ(z_{t:t+H}, φ)`，其中 ρ 为 STL 鲁棒性。使用动力学模型预测未来状态。
  - **学习策略**：用于接触丰富或建模困难的行为，但仅由 STL 监控（不直接优化）。
  - **历史日志**：存储最近 L 条记录（观测、状态、子任务描述、执行器、STL 鲁棒值、完成/失败事件）。当鲁棒性低于阈值时触发 recall，将历史日志传递给 System 2 进行重新规划。
- **STL 编译器**：将技能模板、参数、全局约束转换为 STL 公式（如 G(Safe(stove)), F[5,8](G(Grasp)), G[0,5](¬Grasp) 等）。

### 算法流程（文字说明）
1. 用户输入自然语言指令和场景观测。
2. System 2（VLM + STL编译器）生成子任务序列和对应的 STL 公式。
3. 逐个执行子任务：System 1 选择执行器（MPC 或学习策略）；每步计算 STL 鲁棒性。
4. 若鲁棒性高于阈值 → 继续执行；若低于阈值 → 触发 recall，System 2 基于历史日志和当前 STL 规范重新规划后续子任务。
5. 若鲁棒性满足子任务完成条件 → 推进到下一子任务或调用 System 2 进行下一步规划（可选）。

## 6. 论文的主要结论与发现

1. **精确语言跟随（RQ1）**：STL 引导的规划在所有约束类型（逻辑、空间、逻辑+空间、时序）上均获得了更高的安全成功率，其中时序约束的提升最大。说明 STL 规范能显著提高机器人对精确时空逻辑指令的满足能力，而不牺牲整体任务性能。
2. **策略切换（RQ2）**：STL 鲁棒性可作为实用的切换信号，在障碍物场景中允许系统利用学习策略的接触操作能力和 MPC 的精确约束满足能力，实现优于单一执行器的性能。
3. **少样本重新规划（RQ3）**：STL 违反轨迹（违反的约束、鲁棒性数值、时间戳）提供了结构化反馈，使 VLM 能够更精准地修复特定参数（如增加接近高度），相比标量 MPC 成本反馈，纠正效率更高（更多失败在两次尝试内修复）。
4. **整体结论**：语言条件机器人应受益于一个**显式的规范层**（STL），将高层推理与低层控制连接起来，而不是仅依赖单一学习策略隐式地满足所有任务要求。该框架提升了精确性、可靠性和可解释性。

## 7. 优点

- **方法创新性**：将信号时序逻辑作为 VLM 与机器人控制之间的**共享形式化接口**，贯穿任务分解、STL 编译、低层控制、监控和重新规划全流程，而非仅用于规范生成或可行性检查。
- **混合执行模型**：根据子任务特性选择 STL-guided MPC（精确约束）或学习策略（接触丰富行为），二者通过 STL 鲁棒性统一监控和切换，兼顾了精确性与表现力。
- **可解释性**：STL 鲁棒性提供了连续满意度度量，能够量化任务进展、检测违规，并在重新规划时提供结构化（而非标量）反馈，使系统行为更透明。
- **实验设计**：三个 RQ 覆盖精确性、切换能力和恢复能力，消融对比公平，且包含真实机械臂平台验证，有实际可行性。
- **系统集成**：包括感知（Segment-Anything + VLM 场景描述）、语言处理、STL 编译器、MPC 控制器、历史日志、Recall 机制，工程实现完整。

## 8. 不足与局限

- **算力与训练细节不透明**：未说明 VLM 调用成本、训练行为克隆策略时的 GPU 资源，难以评估可复现性。
- **实验规模有限**：仅在单一桌面操作平台（UR3e）上评估，任务种类和复杂度有限（9个任务，每组实验重复次数未明确给出）。缺乏在杂乱场景、动态环境、多机器人或长时域任务上的验证。
- **学习策略未被 STL 直接优化**：论文承认仅 MPC 直接优化 STL 鲁棒性；学习策略在推理时不能采用 STL 作为控制信号，监控仅用于检测违规，不改进动作生成。
- **高层计划的表达能力**：目前使用顺序子任务列表，无法表达分支、循环等复杂行为。若需要条件性计划需扩展为自动机。
- **系统组件简单**：MPC 求解器、抓取姿态选择、执行循环较简单，在杂乱场景中可能失效。感知依赖固定单次 RGB-D 捕获，不能处理物体移动后更新的动态场景（虽然提及 relocalization，但实时性未验证）。
- **VLM 仍可能产生错误规范**：当场景描述缺少精细空间细节时，VLM 可能生成不完整或不正确的 STL 规范。依赖 LLM 的翻译质量，未提供安全保障机制（如证明正确性）。
- **缺少与纯数据驱动 VLA 方法的直接比较**：基线 VLM-MPC-COST 仍基于 MPC，未与纯端到端 VLA 模型（如 RT-2 等）对比 STL 对约束遵循的影响。
- **未报告统计显著性和误差条**：成功率报告未给出置信区间或多次重复的标准差，说服力有限。
