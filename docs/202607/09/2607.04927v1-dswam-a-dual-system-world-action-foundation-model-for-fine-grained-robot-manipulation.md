## 摘要
世界动作模型（WAMs）通过使用基于视频的世界建模作为机器人动作学习的密集监督，为视觉-语言-动作（VLA）策略提供了一种有前景的替代方案。现有的WAM在物理根基执行方面表现出色，但通常缺乏基于VLM的VLA中用于分解粗粒度指令的显式语言级规划接口。当家庭任务涉及复杂的多步骤目标时，这种分解变得重要，因为粗粒度的用户命令需要被转换为细粒度可执行的子任务序列。同时，该领域仍然缺乏VLA和WAM执行能力之间的公平真实机器人对比，因为现有系统通常在数据、机器人硬件和任务协议上有所不同。为了解决分解差距和受控WAM-VLA对比的需要，我们提出了DSWAM，一种面向精细机器人操作的双系统世界动作基础模型。DSWAM将系统1 WAM执行器作为默认控制路径，并且仅在任务分解有用时，选择性地激活系统2视觉-语言子任务规划器。规划器从短期视觉历史和全局任务提示中预测可执行的子任务，而WAM执行器为每条指令或子任务进行世界感知的动作生成。执行器通过动作预测和视频联合训练进行训练，但推理时直接预测动作块，无需显式生成未来视频。为了使这一执行路径在真实机器人上实用，我们进一步集成了TensorRT加速、异步执行和实时分块（RTC），使得策略查询不会阻塞机器人控制。为了提供与VLA策略的公平真实机器人对比，我们在DeMaVLA真实世界可变形操作设置下构建和评估了DSWAM，并匹配了机器人平台、预训练数据、后训练数据和评估标准。

## Abstract
World Action Models (WAMs) provide a promising alternative to Vision-Language-Action (VLA) policies by using video-based world modeling as dense supervision for robot action learning. Existing WAMs excel at physically grounded execution, but typically lack the explicit language-level planning interface in VLM-based VLAs for decomposing coarse instructions. Such decomposition becomes important when household tasks involve complex multi-step goals, where coarse user commands need to be converted into sequences of fine-grained executable subtasks. Meanwhile, the field still lacks a fair real-robot comparison between VLA and WAM execution capabilities, since existing systems often differ in data, robot embodiments, and task protocols. To address both the decomposition gap and the need for a controlled WAM-VLA comparison, we introduce DSWAM, a Dual-System World Action Foundation Model for fine-grained robot manipulation. DSWAM keeps a System 1 WAM executor as the default control path and optionally activates a System 2 vision-language subtask planner only when task decomposition is useful. The planner predicts executable subtasks from short-term visual history and a global task prompt, while the WAM executor performs world-aware action generation for each instruction or subtask. The executor is trained with action prediction and video co-training, but inference directly predicts action chunks without explicit future video generation. To make this execution path practical on real robots, we further integrate TensorRT acceleration, asynchronous execution, and real-time chunking (RTC) so that policy queries do not block robot control. To provide a fair real-robot comparison with VLA policies, we build and evaluate DSWAM under the DeMaVLA real-world deformable manipulation setting with matched robot platform, pretraining data, post-training data, and evaluation criteria.

- **核心问题**：当前的机器人世界动作模型（WAM）在物理执行层面表现强劲，但缺少类似视觉-语言-动作（VLA）模型中那种用于分解粗糙指令的显式语言级规划能力，难以处理需要多步推理的家庭任务；同时，该领域缺乏在相同的硬件、数据和任务协议下对WAM与VLA执行能力的公平对比。
- **整体含义**：DSWAM提出了一种双系统架构，将语义级任务分解从物理执行中解耦，使WAM执行器作为默认路径，仅在必要时激活一个可选的视觉-语言规划器，从而在不牺牲执行效率的前提下，弥补任务规划的空白，并为WAM与VLA的公平比较提供了受控实验。

## 2. 方法论

### 2.1 核心思想
- 采用双系统设计（System 1 & System 2）。  
  - **System 1（WAM执行器）**：默认的控制路径，基于当前多模态观测直接预测连续动作块，训练时借助视频联合训练学习世界感知的表征，但推理时无需显式生成未来视频。  
  - **System 2（视觉-语言子任务规划器）**：可选模块，仅在指令需要分解时触发，从短期视觉历史和全局任务提示中预测可执行的子任务指令，然后将子任务依次交给System 1执行。

### 2.2 关键技术细节
- **System 2 规划器**  
  - 基于Rynnbrain4B风格的视觉-语言模型。  
  - 输入：最近5帧（采样频率1 Hz）的视觉历史 \(X_t\) 和全局任务提示 \(p\)。  
  - 输出：下一个子任务指令 \(s_t\)；当到达子任务边界时，输出下一个子任务标签或“done”。  
  - 训练数据：从机器人轨迹视频中划分子任务片段，利用滑动窗口构建监督信号，损失函数为自回归语言建模目标 \(L_\text{plan}\)。  
  - 耦合机制：部署时规划器每2秒接收最新5帧，返回子任务指令，System 1据此执行，直至下一次规划更新。

- **System 1 WAM执行器**  
  - 输入：当前多视角图像、当前语言条件（可能来自System 2）和本体感受。  
  - 编码：通过视觉骨干和跨模态编码得到潜在世界特征 \(z_t\)。  
  - 动作生成：基于条件流匹配（conditional flow matching）直接从 \(z_t\) 预测连续动作块 \(A_t = [a_t, \dots, a_{t+H-1}]\)，不生成未来帧。  
  - 训练目标：联合优化动作预测损失 \(L_\text{act}\) 和视频协同训练损失 \(L_\text{vid}\)，两者都采用流匹配的均方误差形式，总损失为 \(L = L_\text{act} + \lambda_\text{vid} L_\text{vid}\)，并保证动作分支无法访问未来视觉令牌。

- **部署优化**  
  - 推理时采用实时分块（RTC）与异步执行，策略预测与机器人控制解耦，避免延迟阻塞。  
  - 使用TensorRT将视觉上下文编码和动作去噪过程分别编译为两个引擎，BF16加速，端到端延迟从198.2 ms降至73.8 ms（约2.69倍提升）。

## 3. 实验设计

## 6. 主要结论与发现
- DSWAM在DeMaVLA真实折叠基准上平均成功率达96.3%，较DeMaVLA（92.5%）提升3.8%，平均完成时间从2分18秒缩短到1分44秒，显著优于VLA策略。  
- 在RoboTwin 2.0模拟任务中，清洁和随机化设置的成功率分别为92.38%和91.90%，均超过所有对比的VLA和WAM基线。  
- 可选的System 2子任务规划器在处理粗粒度指令时能明显提升成功率（71.4%→100%）并减少错误次数，证明了任务分解的价值。  
- TensorRT加速和异步RTC使推理延迟降低约2.69倍，并改善了真实折叠任务的执行稳定性，不改变策略本身即可受益。

## 7. 优点
- **架构设计的合理性**：将物理执行与高层规划解耦，系统1和系统2可选择组合，兼顾效率与灵活性。  
- **公平对比实验**：在完全相同的数据、硬件和任务协议下，直接对比WAM执行模式与强VLA基准，提供了少见的受控实证。  
- **高效部署**：结合训练时期的视频协同学习与推理时的直接动作预测，并引入TensorRT和RTC，保证了真实机器人上的低延迟和稳定控制。  
- **任务覆盖广泛**：模拟和真实实验跨度大，从双臂操作到衣物折叠，验证了模型的通用性。
