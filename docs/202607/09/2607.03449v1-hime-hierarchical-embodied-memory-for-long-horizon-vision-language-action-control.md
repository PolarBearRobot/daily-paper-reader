## 摘要
当前的视觉-语言-动作（VLA）模型在机器人操作方面表现出色，但由于依赖即时观测，常常在需要长期记忆和推理的非马尔可夫任务中遇到困难。现有解决方案面临“频率-能力悖论”，即推理能力更强的模型对于实时控制来说太慢，而更快的模型则缺乏足够的推理能力。为了解决这种架构不匹配，我们提出了HiMe，一个分层具身记忆框架，它将具身智能解耦为用于执行的高频执行器、用于工作记忆的哨兵和用于长期策略的规划器。我们还引入了一个基于跨模态语义图式和主动管理机制的动态知识系统，使机器人能够通过“添加、更新和删除”操作保持记忆可塑性。这种分层设计有效地平衡了实时执行与慢思考规划之间的冲突，显著提高了长时程任务的成功率。实验表明，该方法不仅优于扁平记忆基线，还展现了基于人类偏好自我纠正其内部知识的新能力。

## Abstract
Current Vision-Language-Action (VLA) models excel at robotic manipulation but often struggle with non-Markovian tasks requiring long-term memory and reasoning due to their reliance on immediate observations. Existing solutions face a ''frequency-competence paradox,'' where stronger reasoning models are too slow for real-time control, while faster models lack sufficient reasoning capabilities. To resolve this architectural misalignment, we propose HiMe, a Hierarchical Embodied Memory framework that decouples embodied intelligence into a high-frequency Executor for execution, a Sentry for working memory, and a Planner for long-term strategy. We also introduce a dynamic knowledge system based on cross-modal semantic schemas and active management mechanisms, allowing robots to maintain memory plasticity through ''Add, Update, and Delete'' operations. This hierarchical design effectively balances the conflict between real-time execution and slow thinking planning, significantly improving success rates in long-horizon tasks. Experiments demonstrate that this approach not only outperforms flat memory baselines but also exhibits the novel ability to self-correct its internal knowledge based on human preferences.

## 2. 论文提出的方法论
### 核心思想
- **分层记忆结构**：借鉴人类认知的多级记忆模型，将具身智能系统划分为：
  - **瞬时记忆（Executor，VLA）**：高频反应式策略，仅根据当前观测和当前子目标生成动作，保证实时物理稳定性。
  - **工作记忆（Sentry，轻量VLM）**：异步监控滑动窗口内的观测序列，识别子任务完成或环境偏差，仅在必要时唤醒规划器。
  - **情景记忆（Planner，重型VLM）**：长期存储和主动管理任务相关的上下文与过程性计划，根据哨兵信号进行低频的全局重规划与记忆更新。
- **动态知识管理**：针对机器人交互中多模态丰富性和环境高动态性，提出“增、删、改”（Add, Update, Delete）的主动记忆管理操作，使记忆能够随任务进展和用户偏好变化而持续演化，避免知识僵化。

### 关键技术细节
- **状态空间定义**：
  - 瞬时记忆 $\mathcal{T}_t$：仅当前观测 $o_t$。
  - 工作记忆 $\mathcal{W}_t$：近期 $h_s$ 帧的滑动窗口观测序列 $\{o_{t-h_s}, \dots, o_t\}$。
  - 情景记忆 $\mathcal{E}_t$：分为上下文记忆 $\mathcal{E}^c_t$（多模态键值存储，含视觉和文本）和过程记忆 $\mathcal{E}^p_t$（带状态标签的语言子目标列表）。
- **分层策略分解**：
  - **执行器 $\pi_e(a_t|o_t,\tau_t)$**：以当前子目标 $\tau_t$ 为条件，预测动作，保持无状态以降低计算负载。
  - **哨兵 $\pi_s$**：每 $n_m$ 步基于 $\mathcal{W}_t$ 判断子目标是否完成，输出触发信号 $u_t \in \{0,1\}$，从而降低规划器的调用频率。
  - **规划器 $\pi_p$**：当 $u_t=1$ 时激活，通过两步运作：
    1. **上下文检索**：对用户指令 $l$ 与工作记忆编码，在上下文记忆中检索最相关条目。
    2. **记忆更新与重规划**：根据检索结果和当前状态，执行显式的添加、更新、删除操作更新 $\mathcal{E}^c$，并生成新的过程计划 $\mathcal{E}^p$，指定下一子目标。
- **实现细节**：
  - Planner 采用 GPT-4o（主实验）或 Qwen3-VL-30B（开源替代）；Sentry 采用 Qwen3-VL-8B；Executor 微调自 $\pi_{0.5}$ 模型。
  - 记忆后端使用向量数据库，文本嵌入采用 OpenAI text-embedding-3，多模态条目以文本向量相似度检索。
  - 工作记忆窗口 $h_s=8$，监控间隔 $n_m=5$，Execution 频率为 2 Hz，预测动作块包含 10 步动作。

## 3. 实验设计
### 任务场景与基准
- 在真实 WidowX-250s 机器人平台上设计了三类长时程桌面操作任务（每方法 20 次试验）：
  - **物体搜索**：主动检查不透明盒子以推断内容，放置物体，并在用户引入新玩具时动态更新记忆；最终根据记忆的偏好取出正确物体。
  - **计数**：阅读视觉食谱，根据用户要求准备相应份数的食材，并在用户提出新偏好后，结合记忆的食谱信息进行个性化配料。
  - **重排列**：先将所有玩具收集到盒子清洗，再根据先前记忆的空间布局和用户放置偏好恢复原位。
- 评估指标：任务进度（完成子任务比例）、每次子任务的平均规划器 API 调用次数、记忆命中率。

### 对比方法
- **Transient Memory**：规划器定期调用，仅接收当前观测（无记忆）。
- **Transient Memory w/ Sentry**：加入哨兵触发，但规划器仍无记忆。
- **Flat Memory**：规划器定期调用，使用滑动窗口最近观测加 FIFO 关键帧队列（类似 MemER）。
- **HiMe w/o Sentry**：使用完整规划器记忆设计，但移除哨兵模块，按固定频率调用规划器。
- **HiMe（本文）**：哨兵动态触发 + 完整分层记忆。
- **Human High-level**：由人工给出正确的子任务描述，作为执行器的理论上限。

## 6. 论文的主要结论与发现
- HiMe 在三个长时程任务中平均任务进度达 90%，显著超过所有基线（如 Flat Memory 的 65%），接近人类 oracle 的性能。
- 哨兵机制通过异步触发将规划器调用次数降低约 3 倍，并因提供时序聚合的工作记忆而提高了规划质量。
- 主动记忆管理（增/删/改）比简单的 FIFO 或无管理记忆更有优势，能保持记忆一致性与精简性，避免早期关键信息被挤出。
- 跨模态记忆（文本+图像）优于单一模态，文本擅长语义和逻辑推理，图像提供空间和识别细节。
- 系统展示了根据人类偏好动态修正内部知识的能力，这是传统检索式记忆所不具备的。

## 7. 优点

- **动态记忆管理**：颠覆了被动记录的方式，引入增删改的主动操作，使机器人知识能够与环境变化和用户偏好保持同步，具备一定“自我进化”能力。
- **多模态语义图式**：将视觉经验与高密度文本描述深度融合，使记忆不仅包含图像，还存储了对象归属、规则和偏好等抽象语义，极大增强了记忆的可查询性。

## 8. 不足与局限
- **实验标准化不足**：评估仅在自行设计的三个真实机器人任务上进行，缺乏标准仿真基准，导致结果难以与更广泛的现有方法直接对比，可复现性也受硬件限制。
